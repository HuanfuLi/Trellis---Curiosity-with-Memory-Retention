Create `app/src/hooks/useConceptDashboard.ts`:

```typescript
export interface ConceptDashboardData {
  anchor: Question | undefined;
  qaChildren: Question[];
  // Posts
  posts: DailyPost[];        // bounded to 5, sorted by generatedAt desc
  totalPostCount: number;
  savedPosts: DailyPost[];   // concept posts that are saved
  likedPosts: DailyPost[];   // concept posts that are liked
  // Flashcards
  flashcards: FlashCard[];   // bounded to 5
  totalFlashcardCount: number;
  dueFlashcardCount: number;
  // Podcasts
  podcasts: DailyPodcast[];  // bounded to 3, status === 'ready', sorted by createdAt desc
  totalPodcastCount: number;
  // Collections
  collectionsMap: Map<string, Collection[]>;  // postId → collections for concept posts
  // Health
  leafState: LeafState | null;
  isLoading: boolean;
}

export function useConceptDashboard(anchorId: string | undefined): ConceptDashboardData
```

Implementation approach:
1. Use `useQuestions()` hook for anchor + Q&A children (matches existing AnchorDetailScreen pattern)
2. Build `qaChildIdSet = new Set(qaChildren.map(q => q.id))` for O(1) membership checks
3. Join posts: `postHistoryService.getPosts().filter(p => p.sourceQuestionIds.some(id => qaChildIdSet.has(id)))` — sort by generatedAt desc, slice(0, 5) for bounded, keep full count
4. Join saved/liked: `engagementService.getSavedPosts().filter(p => allConceptPostIds.has(p.id))` (use Set of all concept post IDs from step 3's unsliced result)
5. Join flashcards: `flashcardService.getAll().filter(c => c.nodeId && qaChildIdSet.has(c.nodeId))` — count due where `nextReviewDate <= today()`
6. Join podcasts: `podcastService.getAll().filter(p => p.status === 'ready' && p.questionIds.some(id => qaChildIdSet.has(id)))` — sort by createdAt desc, slice(0, 3)
7. Join collections: for each concept post (use bounded list), call `collectionService.getPostCollections(postId)`, build Map
8. Compute leaf state: `computeLeafState(anchor, qaChildren)`
9. Use `useState` + `useCallback` for a `recompute` function that rebuilds all data
10. Use `useEffect` to subscribe to 6 event types, each calling `recompute()`. Return cleanup function.
11. Use `useEffect` with `[anchorId, questions]` dependency to recompute when anchor or questions change.

CRITICAL: The join for posts goes through Q&A children IDs, NOT anchor IDs. `DailyPost.sourceQuestionIds` contains Q&A node IDs. This is documented in R8 of 51-RESEARCH.md.

CRITICAL: All service reads are synchronous localStorage reads — no async, no loading spinners needed for the data itself. `isLoading` comes from `useQuestions().isLoading`.
