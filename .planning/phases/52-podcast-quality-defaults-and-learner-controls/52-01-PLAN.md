Define prompt building blocks as module-level constants:

**BASE_INSTRUCTION:**
```
You are creating a spoken learning podcast recap. Write ONLY the words to be spoken aloud — no stage directions, no music cues, no markdown formatting, no section headers.
```

**SECTION_INSTRUCTION** (used by standard + deep):
```
Structure the podcast with these sections (use natural spoken transitions between them, do NOT announce section names):
1. RECAP: Summarize what the learner studied for each concept
2. CONNECTIONS: Link concepts to each other or to broader ideas the learner may know
3. MISCONCEPTION CHECK: Address one common misunderstanding per key concept
4. RETRIEVAL QUESTIONS: Ask the listener 2-3 recall questions — pause briefly after each so they can think
5. NEXT ACTION: Suggest what the learner should do next (review weak concepts, explore related topics, practice recall)
```

**BRIEF_SECTION_INSTRUCTION** (abbreviated for brief):
```
Structure the podcast as: a concise recap of each concept, then 1-2 retrieval questions for the listener to pause and answer, then a brief next-step suggestion.
```

**LENGTH_MAP** (Record<PodcastLength, string>):
- brief: `"Keep it concise: approximately 150 words (~60 seconds spoken). Focus on recap and retrieval questions. Keep connections and misconceptions to brief mentions only if critical."`
- standard: `"Target approximately 225 words (~90 seconds spoken). Cover all five sections with balanced depth."`
- deep: `"Target approximately 450 words (~3 minutes spoken). Expand connections across concepts, include 2+ misconception checks, and ask 4-5 retrieval questions with pauses."`

**STYLE_MAP** (Record<PodcastStyle, string>):
- focused: `"Use a structured, clear delivery. Be precise and direct. Minimal asides or filler."`
- conversational: `"Use a warm, natural radio-host style with smooth transitions. Make it engaging but stay educational — no entertainment filler."`
- review: `"Emphasize active recall throughout. After each concept recap, immediately pose a retrieval question. End with a rapid-fire self-test covering all concepts."`

**COVERAGE_CONSTRAINT:**
```
IMPORTANT: You MUST cover ALL of the following concepts in the podcast. Do not skip any concept.
```

Implement `buildPodcastPrompt(conceptLines: string[], options: PodcastOptions): { system: string; user: string }`:

1. Start with BASE_INSTRUCTION
2. Append SECTION_INSTRUCTION (standard/deep) or BRIEF_SECTION_INSTRUCTION (brief)
3. Append LENGTH_MAP[options.length]
4. Append STYLE_MAP[options.style]
5. Append COVERAGE_CONSTRAINT
6. Join with double newlines
7. User message: `"Concepts to cover:\n"` + conceptLines.map(l => `"- ${l}"`).join('\n')
8. Return `{ system, user }`

Implement `computeOptionsHash(conceptIds: string[], locale: string, options: PodcastOptions): string`:

Simple deterministic string — no crypto needed:
```typescript
export function computeOptionsHash(
  conceptIds: string[],
  locale: string,
  options: PodcastOptions,
): string {
  return JSON.stringify({ conceptIds, locale, length: options.length, style: options.style });
}
```

Do NOT sort conceptIds — order matters (different orderings produce different podcast flows). The caller is responsible for consistent ordering if they want cache hits.

CRITICAL: The system prompt is for the LLM generating the podcast SCRIPT. It stays in English (per CLAUDE.md — LLM system prompts stay English; applyLocaleDirective handles response language). Do NOT translate prompt building blocks.
