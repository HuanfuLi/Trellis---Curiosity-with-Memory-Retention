## What Shipped

**Phase 20 — Orchestration & Diagnostic Dialogue (ORCH-01..03, DIAG-01..03, PORTAL-01..03)**
- `OrchestrationStrategy` interface on top of `trajectoryAnalyzerService`
- Feed + Planner scorer consume strategy hints (retrieval vs discovery mode)
- Multi-turn Socratic diagnostic check-in with confusion/curiosity/confidence signal updates
- `PortalCard` replaces flat planner suggestions (topic, description, flashcard/post/question quick-links)

**Phase 21 — Review Cap Fix + Generate-on-Enter Posts (REVIEW-01..05, POST-01..08)**
- Daily-cap → daily-goal progress bar (50 default, no hard limit)
- Card-face-only batch feed generation; essay body streams on PostDetailScreen entry
- Pre-built UI shell eliminates layout shift during streaming
- Cached essays persist across re-visits; video/news posts defer LLM summary to on-enter

**Phase 22 — Swipe Navigation (SWIPE-01..10)**
- Horizontal swipe between 5 top-level screens with axis lock, 20% threshold, rubber-band edges
- All 5 screens always-mounted; GraphScreen eager-loaded
- Bottom nav highlight interpolates with drag; keyboard-open + nested-draggable suppression
- Sub-screens render via Outlet overlay; **Phase 26 later reverted tap-to-animated-jump back to instant transport** (ADDENDUM)

**Phase 23 — Incremental Classification + Rate Limiter (PIPE-01..07, RATE-01..06, REORG-01..05, DEDUP-01..05)**
- 3-step branch→cluster→anchor pipeline with append-only KV-cache-friendly threading
- Short-circuit on NEW; retry-then-fall-back to single-call classification
- Monthly rate limiter on askStreaming (off by default; 80% warning, 100% hard block)
- Provider `jsonMode` enforcement + three-tier parser (direct/balanced/repair)
- Symmetric labelKey dedup + NEW→existing coercion (Plan 05 deferred, not executed)

**Phase 24 — Retroactive Verification & Doc Gap Closure (DIAG-02, PORTAL-01..03, POST-02)**
- Generated missing 20-04-SUMMARY.md, 21-03-SUMMARY.md
- Created Phase 20 + 21 VERIFICATION.md files (both human_needed)
- Completed Phase 23 Nyquist validation
- Deferred out-of-scope requirements to v1.2 (REQUIREMENTS.md)

**Phase 25 — Anime Knowledge Tree (Trellis) (PHASE-25-WAVE-0..POLISH)**
- Ghibli-aesthetic trellis hero on PlannerScreen visualizes review health as vine garden
- Data layer with seeded mulberry32 PRNG, vine/leaf layout, worst-child-wins aggregation
- Three switchable rendering variants (A/C/V — V later removed per user); blossom/fruit lifecycle
- Route-aware sway gate; end-to-end review→leaf smoke test; manual QA archived

**Phase 26 — Trellis Harvest + Node Actions + Suggested Moves Refactor (D-01..D-23)**
- 3-column status panel (Dying | Fruit | Dead) with harvest button (fly-to-counter + confetti + credits)
- Heal (review+podcast), re-plant (schedule reset+post), prune (archive+animation) actions on Suggested Moves rows

**Phase 27 — i18n/L10n Support (D-01..D-24)**
